# Story 1.4: Basic Prospect List UI

## Status: Ready for Review

## Story

> As a consultant,
> I want to see my prospects in a simple web interface,
> so that I can verify the scraping results and browse my leads.

## Epic

Epic 1: Foundation & Data Pipeline

## Acceptance Criteria

- [ ] 1. Página `/` lista prospects em tabela com colunas: nome, cargo, empresa, status, data
- [ ] 2. Server Components para data fetching (SSR)
- [ ] 3. Filtro por status (dropdown) funcional
- [ ] 4. Campo de busca por texto (nome, empresa) funcional
- [ ] 5. Clicar em prospect navega para `/prospects/[id]` com detalhes completos
- [ ] 6. Página `/settings` com formulário de busca (cargos, empresas, localização) + botão "Buscar Prospects"
- [ ] 7. Feedback visual durante scraping (loading state)
- [ ] 8. Layout base com navegação (shadcn/ui) e paleta neutra/corporativa

## Dev Notes

### Rendering Strategy [Source: architecture.md#8.1]

- `/` (Dashboard) — Server Component + Client islands
- `SearchFilters` — Client Component (`'use client'`) para interatividade
- `ScrapeForm` — Client Component com polling de status

### Components [Source: architecture.md#3]

```
components/
├── prospects/
│   ├── prospect-table.tsx      # Table view (Server Component friendly)
│   ├── prospect-profile.tsx    # LinkedIn data display
│   └── search-filters.tsx      # Client Component: status + text filters
```

### Search Filters (Client Component)

- Usar URL search params (`useSearchParams`) para filtros
- Debounce no campo de busca (300ms)
- Filtro de status: dropdown com opções: all, new, message_generated, sent, replied, converted, discarded

### Scrape Form (`/settings`)

- Formulário com campos: titles (textarea, um por linha), companies (textarea), locations (textarea), keywords (textarea), maxResults (number, default 50)
- Botão "Buscar Prospects" → chama `POST /api/scrape`
- Polling visual: progress bar ou spinner enquanto job está running
- Resultado: "X prospects encontrados, Y novos adicionados"

### Branding [Source: prd.md#3.5]

- Paleta: cinza (#f8f9fa, #e9ecef, #6c757d), branco (#fff), navy (#1e3a5f)
- Accent: azul para CTAs
- shadcn/ui theme customizado com essas cores
- Tipografia: Inter ou system sans-serif

### Prospect Detail Page

> **Escopo incremental:** Esta story cria `/prospects/[id]` como versão básica (perfil + link LinkedIn). Story 2.3 adiciona MessageGenerator + InteractionTimeline. Story 3.3 completa com quick actions, navegação e profile detalhado.

- Página `/prospects/[id]` — Server Component
- Mostra todos os campos do prospect: nome, cargo, empresa, setor, headline, resumo, localização
- Link para LinkedIn (abre nova aba)
- Placeholder para interações (será implementado em stories posteriores)
