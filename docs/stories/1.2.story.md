# Story 1.2: Database Schema & Prospect Model

## Status: Draft

## Story

> As a consultant,
> I want prospects stored in a structured database,
> so that I can query and manage my lead pipeline efficiently.

## Epic

Epic 1: Foundation & Data Pipeline

## Acceptance Criteria

- [ ] 1. Turso database configurado com Drizzle ORM (libSQL driver)
- [ ] 2. Tabela `prospects` com campos: id, name, title, company, industry, linkedin_url (unique), headline, summary, location, profile_image_url, status (enum), raw_data, created_at, updated_at
- [ ] 3. Tabela `interactions` com campos: id, prospect_id (FK cascade), type (enum), content, template_id (FK nullable), metadata (JSON), created_at
- [ ] 4. Tabela `message_templates` com campos: id, name, type (enum), system_prompt, user_prompt_template, is_active, created_at, updated_at
- [ ] 5. Tabela `scrape_jobs` com campos: id, apify_run_id, criteria (JSON), status (enum), prospects_found, prospects_new, error, created_at, completed_at
- [ ] 6. Migrations versionadas via Drizzle Kit
- [ ] 7. Seed com 3 templates default (Connection Request, Follow-up, InMail)
- [ ] 8. Route Handlers CRUD para prospects: `GET/POST /api/prospects`, `GET/PATCH/DELETE /api/prospects/[id]`
- [ ] 9. Filtros na listagem: status, company, search (query params)
- [ ] 10. Testes unitários para operações de banco

### Turso Connection [Source: architecture.md#4.1]

```typescript
// lib/db/index.ts
import { drizzle } from 'drizzle-orm/libsql';
import { createClient } from '@libsql/client';
const client = createClient({
  url: process.env.TURSO_DATABASE_URL!,
  authToken: process.env.TURSO_AUTH_TOKEN,
});
export const db = drizzle(client, { schema });
```

### Drizzle Config

```typescript
// drizzle.config.ts
export default {
  schema: './lib/db/schema.ts',
  out: './drizzle/migrations',
  dialect: 'sqlite',
  driver: 'turso',
  dbCredentials: {
    url: process.env.TURSO_DATABASE_URL!,
    authToken: process.env.TURSO_AUTH_TOKEN,
  },
};
```

### API Endpoints [Source: architecture.md#5.1]

```
GET    /api/prospects?status=X&search=Y&page=1  → { data: Prospect[], total: number }
POST   /api/prospects                            → Prospect
GET    /api/prospects/[id]                       → Prospect (with interactions)
PATCH  /api/prospects/[id]                       → Prospect
DELETE /api/prospects/[id]                       → { deleted: true }
```

### Error Format [Source: architecture.md#10.1]

```json
{ "error": { "code": "PROSPECT_NOT_FOUND", "message": "...", "statusCode": 404 } }
```

### Seed Templates [Source: prd.md#FR4]

3 templates default: Connection Request, Follow-up, InMail. Cada um com system_prompt e user_prompt_template pré-configurados para o tom "Strategic Reset" (consultor sênior, não vendedor).
