# Story 1.3: Apify Integration & LinkedIn Scraping

## Status: Ready for Review

## Story

> As a consultant,
> I want to search for prospects on LinkedIn using configurable criteria,
> so that I can build a qualified lead list matching my ICP.

## Epic

Epic 1: Foundation & Data Pipeline

## Acceptance Criteria

- [ ] 1. Integração com Apify REST API v2 via fetch usando `APIFY_TOKEN`
- [ ] 2. Route Handler `POST /api/scrape` aceita parâmetros: titles, companies, locations, keywords, maxResults
- [ ] 3. Sistema chama Actor do Apify Store via REST API para LinkedIn scraping
- [ ] 4. Resultados são parseados e salvos na tabela `prospects` (deduplicação por linkedin_url)
- [ ] 5. Scrape job registrado na tabela `scrape_jobs` com status tracking
- [ ] 6. Route Handler `GET /api/scrape/[jobId]` para polling de status
- [ ] 7. Tratamento de erros: token inválido, Actor indisponível, timeout
- [ ] 8. Testes para parsing de dados do Apify e deduplicação

## Dev Notes

### Apify REST API Client [Source: architecture.md#6.1]

```typescript
// lib/apify.ts — REST API client (NO SDK, usar fetch)
const APIFY_BASE = 'https://api.apify.com/v2';

// 3 funções:
// runActor(actorId, input) → POST /acts/{actorId}/runs
// getRunStatus(runId) → GET /actor-runs/{runId}
// getDatasetItems(datasetId) → GET /datasets/{datasetId}/items
// Header: Authorization: Bearer ${APIFY_TOKEN}
```

### Scraping Flow [Source: architecture.md#6.1]

1. `POST /api/scrape` → chama `runActor()` → salva job com status `running`
2. Frontend faz polling `GET /api/scrape/[jobId]` → chama `getRunStatus()`
3. Quando `status === 'SUCCEEDED'` → chama `getDatasetItems()` → parse + dedupe + save
4. Job atualizado para `completed` com contadores

### Request Format [Source: architecture.md#5.2]

```json
{
  "titles": ["CEO", "CTO", "VP", "Diretor"],
  "companies": ["Itaú", "Bradesco", "Magazine Luiza"],
  "locations": ["Brazil"],
  "keywords": ["transformação digital", "inovação"],
  "maxResults": 50
}
```

### Deduplication Logic

- Deduplicar por `linkedin_url` (campo unique na tabela prospects)
- Se prospect já existe, NÃO atualizar — manter dados originais
- Contar `prospects_found` (total) e `prospects_new` (inseridos)

### Actor Recomendado

`anchor/linkedin-people-search-scraper` ou similar do Apify Store. O actorId deve ser configurável (pode ser env var ou parâmetro).
