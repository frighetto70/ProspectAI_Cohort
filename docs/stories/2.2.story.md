## Dev Notes

### Default Templates (Seed)

> **Nota:** O seed básico com 3 templates é criado em Story 1.2 (Task 7). Esta story enriquece os templates com system_prompt e user_prompt_template completos para uso com Gemini.

**Connection Request** (type: connection_request):
- system_prompt: Tom de colega estratégico, máx 300 caracteres, mencionar algo do perfil
- user_prompt_template: Dados do prospect + "Gere uma mensagem de connection request curta e direta"

**Follow-up** (type: follow_up):
- system_prompt: Tom de follow-up após conexão aceita, mais detalhado (até 500 chars)
- user_prompt_template: Dados do prospect + "Gere uma mensagem de follow-up mencionando o Strategic Reset"

**InMail** (type: inmail):
- system_prompt: Tom mais formal, até 1000 chars, pode incluir proposta de valor
- user_prompt_template: Dados do prospect + "Gere um InMail profissional com proposta de valor clara"

### API Endpoints [Source: architecture.md#5.1]

```
GET    /api/templates          → Template[]
POST   /api/templates          → Template
PATCH  /api/templates/[id]     → Template
DELETE /api/templates/[id]     → { deleted: true }
```

### Template Variables

Variáveis são interpoladas NO prompt enviado ao Gemini, não no template em si. O template contém os placeholders `{{var}}` que são substituídos antes de chamar a API.

### Templates Page [Source: architecture.md#3]

```
app/templates/page.tsx         # Server Component: lista templates
components/messages/
└── template-selector.tsx      # Client Component: dropdown de seleção
```

- Lista de templates com nome, tipo, status (ativo/inativo)
- Formulário de edição inline ou modal
- Preview: mostrar como ficam as variáveis substituídas com dados mock
