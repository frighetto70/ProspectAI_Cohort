# Story 2.3: Message Review & Approval UI

## Status: Draft

## Story

> As a consultant,
> I want to review, edit, and approve AI-generated messages before sending,
> so that I maintain quality control over every outreach.

## Epic

Epic 2: AI Message Generation

## Acceptance Criteria

- [ ] 1. Na página `/prospects/[id]`, layout side-by-side: perfil do prospect (esquerda) + mensagem gerada (direita)
- [ ] 2. Mensagem editável inline com textarea
- [ ] 3. Botão "Regenerar" para gerar nova variação sem perder a anterior
- [ ] 4. Botão "Copiar" que copia mensagem para clipboard com feedback visual (toast)
- [ ] 5. Seletor de template visível na tela de geração
- [ ] 6. Geração em lote: selecionar múltiplos prospects na listagem e gerar mensagens para todos
- [ ] 7. Histórico de mensagens geradas visível na página do prospect

### Message Generator (Client Component)

> **Escopo incremental:** `/prospects/[id]` foi criada como placeholder em Story 1.4. Esta story adiciona MessageGenerator (side-by-side) + InteractionTimeline. Story 3.3 completa com quick actions, navegação e profile detalhado.

- Layout: grid 2 colunas em desktop, stack em mobile
- Esquerda: ProspectProfile (dados do prospect)
- Direita: MessageEditor + TemplateSelector + botões (Gerar, Regenerar, Copiar)
- Estado: `generatedMessage`, `isGenerating`, `selectedTemplateId`

### Message Editor (Client Component)

- Textarea com a mensagem gerada, editável
- Contador de caracteres (connection request tem limite ~300)
- Botão "Copiar" → `navigator.clipboard.writeText()` → toast "Mensagem copiada!"

### Bulk Generation

> **Nota de escopo:** Bulk generation (AC6, Tasks 9-11) é uma feature complexa. Se o escopo desta story ficar grande demais durante implementação, considerar extrair para Story 2.4.

- Na página `/` (tabela de prospects), adicionar checkboxes de seleção
- Botão "Gerar Mensagens" (aparece quando >= 1 selecionado)
- Chama `POST /api/prospects/bulk-generate` com `{ prospectIds: number[], templateId: number }`
- Route Handler processa sequencialmente com throttle (1s entre cada)
- Feedback: "Gerando mensagem X de Y..."

### Bulk Generate Route Handler [Source: architecture.md#5.1]

```
POST /api/prospects/bulk-generate → { queued: number }
```

### Interaction History

> **Nota:** `interaction-timeline.tsx` é criado nesta story (Task 8) e reutilizado/estendido em Story 3.3.

- Na página `/prospects/[id]`, abaixo do generator, mostrar timeline das interações
- Usar `components/interaction-timeline.tsx`
- Ordenar por created_at desc
- Cada item: tipo (badge), conteúdo, data
